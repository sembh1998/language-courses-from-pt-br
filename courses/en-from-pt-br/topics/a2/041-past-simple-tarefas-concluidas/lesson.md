# Past simple: tarefas concluídas

Nível: A2

## Objetivo

Relatar tarefas profissionais concluídas usando o past simple com verbos regulares e irregulares frequentes, marcadores de tempo e uma sequência clara de ações.

## Explicação

Use o **past simple** para ações concluídas em um momento passado: `I fixed the
bug yesterday`, `We sent the report last Friday`. O marcador de tempo ajuda a
mostrar que a ação terminou: `yesterday`, `last week`, `two hours ago`, `on
Monday`.

Verbos regulares recebem **-ed**: `fix → fixed`, `create → created`, `deploy →
deployed`. Observe ajustes de escrita: `plan → planned`, `study → studied`,
`close → closed`. A pronúncia de **-ed** varia, mas a forma escrita permanece a
mesma.

Verbos irregulares têm formas próprias e precisam ser aprendidos em contexto:
`write → wrote`, `send → sent`, `run → ran`, `find → found`, `make → made`,
`read → read`. Em `read`, a escrita não muda, mas no passado a pronúncia é
semelhante a “red”.

Para um update curto, use sujeito + verbo no passado + complemento: `Ana wrote
the migration notes.` Para organizar várias ações, use `first`, `then`, `after
that` e `finally`: `First, I ran the tests. Then, I fixed two errors.`

Não use `did` em frases afirmativas comuns: `I sent the file`, não `I did sent
the file`. Perguntas e negativas com `did` serão tratadas no próximo tópico.

## Exemplos

| Inglês | Tradução | Observação |
|---|---|---|
| I fixed the login bug yesterday. | Eu corrigi o bug de login ontem. | Verbo regular com -ed e marcador de tempo. |
| We created a new test last week. | Nós criamos um novo teste na semana passada. | Create termina em e, então recebe apenas -d. |
| Marta wrote the release notes. | Marta escreveu as notas da versão. | Write tem a forma irregular wrote. |
| The team ran the full test suite on Friday. | A equipe executou a suíte completa de testes na sexta-feira. | Run muda para ran no passado. |
| First, I found the cause. Then, I updated the config. | Primeiro, encontrei a causa. Depois, atualizei a configuração. | Sequência de duas tarefas concluídas. |

## Erros comuns

- Usar o infinitivo em uma afirmação passada: `Yesterday I fix the bug` deve ser `Yesterday I fixed the bug`.
- Regularizar um verbo irregular: use `wrote`, não `writed`; `sent`, não `sended`.
- Usar did com o verbo no passado em afirmações: `I did sent` não é a forma neutra; use `I sent`.
- Esquecer a duplicação em planned: `plan → planned`.
- Misturar presente e passado no mesmo relato sem motivo.

## Prática guiada

Transforme para o passado:

- `I fix the issue every day.` → `I fixed the issue yesterday.`
- `We run the tests every morning.` → `We ran the tests this morning.`
- `She writes the update.` → `She wrote the update yesterday.`

Mini-update:

`First, I reviewed the ticket. Then, I found the problem. Finally, I sent the fix.`

## Resumo

O past simple relata ações concluídas. Verbos regulares usam -ed; verbos irregulares têm formas próprias. Marcadores de tempo e conectores ajudam a transformar ações isoladas em um update profissional claro.
