# Verbos de ação em desenvolvimento

Nível: A1

## Objetivo

Usar verbos comuns de desenvolvimento em frases afirmativas, negativas e perguntas simples.

## Explicação

Muitos status técnicos usam verbos curtos: `write`, `read`, `review`, `merge`, `push`, `pull`, `build`, `test`, `fix`, `update` e `install`.

No presente simples, use a forma básica com `I`, `you`, `we` e `they`: `I push code.` Em negativas, use `don't`: `I don't push to main.` Em perguntas, use `Do`: `Do you review code?`

Com `he`, `she` e `it`, acrescentamos `s` na afirmativa: `She reviews code.` Em perguntas e negativas, use `does` ou `doesn't` e mantenha o verbo principal na forma básica: `Does she review code?`, `She doesn't review code.`

## Exemplos

| Inglês | Tradução | Observação |
|---|---|---|
| I write code every day. | Eu escrevo código todos os dias. | Rotina com `I`. |
| We review pull requests. | Nós revisamos pull requests. | `we` usa verbo básico. |
| She fixes bugs. | Ela corrige bugs. | Com `she`, use `fixes`. |
| They don't merge today. | Eles não fazem merge hoje. | Negativa com `don't`. |
| Do you install packages? | Você instala pacotes? | Pergunta com `Do you...?`. |

## Erros comuns

Evite `She fix bugs`. O correto é `She fixes bugs`.

Evite `Do she push code?`. Com `she`, use `Does she push code?`

Evite `I no install packages`. Use `I don't install packages`.

## Prática guiada

Use o padrão: sujeito + verbo + objeto. Depois crie a negativa com `don't` ou `doesn't` e a pergunta com `Do` ou `Does`.

## Resumo

Verbos de ação ajudam a explicar trabalho técnico com frases curtas. Para A1, domine afirmativas, negativas e perguntas simples.
