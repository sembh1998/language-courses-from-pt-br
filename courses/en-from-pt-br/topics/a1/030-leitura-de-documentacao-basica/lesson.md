# Leitura de documentação básica

Nível: A1

## Objetivo

Reconhecer partes comuns de documentação em inglês e encontrar comandos rapidamente usando leitura por varredura.

## Explicação

Documentação técnica costuma ter títulos, listas e blocos de comandos. No A1, você não precisa entender cada palavra. Procure palavras-chave:

- `Overview` = visão geral
- `Requirements` = requisitos
- `Installation` = instalação
- `Usage` = uso
- `Warning` = aviso importante
- `Note` = nota ou observação

Para achar comandos, procure linhas com palavras como `npm`, `git`, `run`, `install` ou símbolos como `$`.

## Exemplos

| Inglês | Tradução | Observação |
|---|---|---|
| Overview | Visão geral | Normalmente explica o projeto. |
| Requirements | Requisitos | Lista o que você precisa antes. |
| Installation | Instalação | Mostra passos para instalar. |
| Usage | Uso | Mostra como usar. |
| Warning: Do not commit secrets. | Aviso: não faça commit de segredos. | `Warning` merece atenção. |
| Note: This command is slow. | Nota: este comando é lento. | `Note` adiciona informação útil. |

## Erros comuns

- Não leia a documentação inteira quando a tarefa é achar um comando. Faça `skim`: passe os olhos procurando termos.
- Não ignore `Warning`; geralmente indica risco.
- Não confunda `requirements` com pedidos informais. Em documentação, são requisitos necessários.

## Prática guiada

Ao abrir uma página de documentação, siga esta ordem:

1. Leia o título principal.
2. Procure `Requirements`.
3. Procure `Installation`.
4. Encontre comandos em listas ou blocos de código.
5. Leia `Warning` e `Note` antes de executar algo.

## Resumo

Documentação básica pode ser lida por partes. Foque em títulos, listas, passos, avisos, notas e comandos.
