# Can/can't para habilidades e permissões

Nível: A1

## Objetivo

Você vai usar `can` e `can't` para falar de habilidades técnicas, pedir ajuda e indicar permissões em sistemas.

## Explicação

Use `can` para dizer que alguém consegue fazer algo ou tem permissão. Use `can't` para dizer que não consegue ou não tem permissão.

| Inglês | Tradução | Observação |
|---|---|---|
| I can fix it. | Eu consigo corrigir isso. | Habilidade. |
| Can you join? | Você pode entrar? | Pedido. |
| We can't deploy now. | Não podemos fazer deploy agora. | Limitação ou permissão. |
| She can access the system. | Ela pode acessar o sistema. | Permissão. |
| They can't edit the file. | Eles não podem editar o arquivo. | Sem permissão. |

Depois de `can` ou `can't`, use o verbo na forma básica: `can fix`, `can join`, `can't deploy`. Não use `to` depois de `can`.

## Exemplos

| Inglês | Tradução | Observação |
|---|---|---|
| I can fix the bug today. | Eu consigo corrigir o bug hoje. | Habilidade técnica. |
| Can you help me? | Você pode me ajudar? | Pedido de ajuda. |
| We can test the app now. | Nós podemos testar o aplicativo agora. | Possibilidade. |
| I can't access the dashboard. | Eu não consigo acessar o painel. | Problema de permissão. |
| Can Ana deploy the update? | A Ana pode fazer deploy da atualização? | Pergunta sobre permissão. |
| The user can't delete files. | O usuário não pode apagar arquivos. | Regra do sistema. |

## Erros comuns

1. Usar `to` depois de `can`: prefira `I can fix it`, não `I can to fix it`.

2. Adicionar `s` ao verbo depois de `can`: prefira `She can access`, não `She can accesses`.

3. Esquecer que `can't` já é negativo: não use `don't can't`.

## Prática guiada

Faça três frases: uma habilidade (`I can...`), uma pergunta (`Can you...?`) e uma limitação (`We can't...`).

## Resumo

`Can` expressa habilidade, possibilidade ou permissão. `Can't` expressa impossibilidade ou falta de permissão. O verbo depois deles fica na forma básica.
