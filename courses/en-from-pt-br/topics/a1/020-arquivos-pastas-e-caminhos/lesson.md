# Arquivos, pastas e caminhos

Nível: A1

## Objetivo

Falar sobre arquivos, pastas e caminhos em inglês, dizendo onde algo está em um projeto.

## Explicação

Use `file` para arquivo e `folder` para pasta. `Path` é o caminho até um arquivo ou pasta. `Root folder` é a pasta principal do projeto. `Config file` é um arquivo de configuração.

Para localização, use preposições simples: `in` para dentro de uma pasta, `under` para abaixo de uma pasta, `at` para um caminho específico.

## Exemplos

| Inglês | Tradução | Observação |
|---|---|---|
| The file is in the root folder. | O arquivo está na pasta raiz. | `in` indica dentro. |
| The config file is under src. | O arquivo de configuração está abaixo de src. | `under` indica hierarquia. |
| The path is /app/config.yaml. | O caminho é /app/config.yaml. | Caminho específico. |
| Open the test folder. | Abra a pasta de testes. | Comando simples. |
| Where is the README file? | Onde está o arquivo README? | Pergunta de localização. |

## Erros comuns

Evite `in /app/config.yaml` quando quiser dizer o caminho exato. Prefira `at /app/config.yaml`.

Evite confundir `file` e `folder`: arquivo tem conteúdo; pasta contém arquivos.

## Prática guiada

Use o padrão `The file is in...`, `The folder is under...` e `The path is...` para descrever um projeto.

## Resumo

Com `file`, `folder`, `path`, `root folder` e `config file`, você consegue pedir e dar localização básica em um projeto.
