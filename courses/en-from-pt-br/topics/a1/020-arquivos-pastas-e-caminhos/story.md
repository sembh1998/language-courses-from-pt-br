# História: Arquivos, pastas e caminhos

Nível: A1

## História em inglês

Tom opens a project.

The root folder has many files.

He looks for the config file.

The file is in the root folder.

The file name is app.yaml.

The tests are under the tests folder.

Tom checks the path and updates the file.

## Tradução linha a linha

| Inglês | Tradução |
|---|---|
| Tom opens a project. | Tom abre um projeto. |
| The root folder has many files. | A pasta raiz tem muitos arquivos. |
| He looks for the config file. | Ele procura o arquivo de configuração. |
| The file is in the root folder. | O arquivo está na pasta raiz. |
| The file name is app.yaml. | O nome do arquivo é app.yaml. |
| The tests are under the tests folder. | Os testes estão abaixo da pasta tests. |
| Tom checks the path and updates the file. | Tom verifica o caminho e atualiza o arquivo. |

## Vocabulário da história

| Inglês | Tradução | Observação |
|---|---|---|
| `root folder` | pasta raiz | Pasta principal do projeto. |
| `config file` | arquivo de configuração | Arquivo com opções do sistema. |
| `file name` | nome do arquivo | Exemplo: `app.yaml`. |
| `under` | abaixo de | Indica hierarquia de pastas. |
| `path` | caminho | Localização do arquivo. |

## Perguntas de compreensão

1. O que Tom abre?
2. O que ele procura?
3. Onde está o arquivo de configuração?
4. Qual é o nome do arquivo?
5. Onde estão os testes?
