# Projeto A1: primeiro dia em um time internacional

Nível: A1

## Objetivo

Neste projeto, você vai simular seu primeiro dia em um time internacional. Você vai se apresentar, ler instruções de setup, pedir ajuda, escrever um update simples e entender uma mensagem de erro curta.

## Explicação

O projeto junta cinco tarefas de comunicação A1.

Primeiro, apresente-se:

`Hi, I am a software developer from Brazil. I work with JavaScript.`

Depois, siga instruções simples:

- `Open the repository.`
- `Install the packages.`
- `Run the tests.`

Se algo falhar, peça ajuda:

`Can you help me? The install failed.`

Para update de status, use frases curtas:

`I installed the packages. The tests failed. I need help with the error.`

## Exemplos

| Inglês | Tradução | Observação |
|---|---|---|
| Hi, I am new on the team. | Oi, eu sou novo no time. | Abertura simples. |
| I work with frontend code. | Eu trabalho com código frontend. | Descreve função. |
| Open the repository. | Abra o repositório. | Instrução de setup. |
| The package is missing. | O pacote está ausente. | Mensagem de erro simples. |
| Can you help me with setup? | Você pode me ajudar com a configuração? | Pedido educado e direto. |
| Today, I read the docs and ran the tests. | Hoje, eu li a documentação e rodei os testes. | Update curto. |

## Erros comuns

1. Tentar escrever um update longo demais.

No A1, prefira três frases curtas.

2. Pedir ajuda sem explicar o problema.

Melhor: `Can you help me? The package is missing.`

3. Ignorar comandos básicos.

Palavras como `open`, `install`, `run` e `check` são centrais no setup.

## Prática guiada

Projeto final:

1. Escreva uma apresentação de duas frases.
2. Leia três instruções de setup.
3. Explique um erro simples.
4. Peça ajuda.
5. Escreva um update de três frases.

## Resumo

Ao final do A1, você deve conseguir participar de um primeiro dia simples: dizer quem é, entender passos básicos, reportar um erro curto e mandar um update compreensível.
