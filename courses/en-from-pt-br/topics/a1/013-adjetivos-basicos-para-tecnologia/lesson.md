# Adjetivos básicos para tecnologia

Nível: A1

## Objetivo

Você vai descrever ferramentas, sistemas e problemas com adjetivos simples como `fast`, `slow`, `new`, `old`, `easy`, `hard`, `simple`, `complex`, `safe`, `public` e `private`.

## Explicação

Em inglês, o adjetivo geralmente vem antes do substantivo: `a fast app`, `a new tool`, `a private file`.

| Inglês | Tradução | Observação |
|---|---|---|
| a fast app | um aplicativo rápido | Adjetivo antes de `app`. |
| a slow system | um sistema lento | Descreve desempenho. |
| an easy task | uma tarefa fácil | Use `an` antes de som de vogal. |
| a hard bug | um bug difícil | `Hard` pode significar difícil. |
| a safe password | uma senha segura | Segurança. |
| a private file | um arquivo privado | Acesso restrito. |

Para dizer que algo é assim, use `is` ou `are`: `The app is fast`, `The files are private`.

## Exemplos

| Inglês | Tradução | Observação |
|---|---|---|
| This is a simple tool. | Esta é uma ferramenta simples. | Adjetivo antes do substantivo. |
| The old server is slow. | O servidor antigo está lento. | Dois adjetivos possíveis na frase. |
| The new dashboard is easy. | O painel novo é fácil. | `New` antes de `dashboard`. |
| This is a complex problem. | Este é um problema complexo. | Problema difícil de entender. |
| The report is public. | O relatório é público. | Sem substantivo depois do adjetivo. |
| The file is private. | O arquivo é privado. | Acesso restrito. |

## Erros comuns

1. Colocar o adjetivo depois do substantivo: prefira `a fast app`, não `an app fast`.

2. Usar plural no adjetivo: em inglês, adjetivo não recebe plural. Use `private files`, não `privates files`.

3. Confundir `hard` com `hardly`. Neste tópico, `hard` significa difícil.

## Prática guiada

Descreva três coisas do seu trabalho: `The app is fast`, `This is a hard task`, `The file is private`.

## Resumo

Adjetivos em inglês ficam antes do substantivo e não mudam no plural. Use-os para descrever velocidade, idade, dificuldade, simplicidade e acesso.
