# Gabarito: Leitura de mensagens de erro simples

Nível: A1

## Gabarito dos exercícios

| ID | Resposta | Explicação |
|---|---|---|
| 1.1 | obrigatório | `Required` indica campo obrigatório. |
| 1.2 | File not found | `Not found` significa não encontrado. |
| 1.3 | warning | `Warning` é aviso ou alerta. |
| 2.1 | failed | `Login failed` significa que o login falhou. |
| 2.2 | Invalid | `Invalid email` indica e-mail inválido. |
| 2.3 | required | `Token is required` indica que o token é obrigatório. |
| 3.1 | Senha ausente. | `Missing` significa ausente ou faltando. |
| 3.2 | Página não encontrada. | `Not found` significa não encontrado. |
| 4.1 | O e-mail é obrigatório; o usuário precisa preencher o e-mail. | A ação sugerida é preencher o campo. |
| 4.2 | O token é inválido; é preciso verificar ou trocar o token. | `Invalid` indica valor incorreto. |
| 5.1 | File not found. | Essa é a mensagem curta padrão para arquivo não encontrado. |

## Gabarito do teste

| ID | Resposta | Explicação |
|---|---|---|
| 1 | A build falhou. | `Failed` indica que a ação não deu certo. |
| 2 | Preencher o nome | `Required` pede que o campo seja preenchido. |
| 3 | invalid | `Invalid` indica formato ou valor inválido. |
| 4 | found | `Not found` forma a expressão não encontrado. |
| 5 | Aviso: o disco está cheio. | `Warning` é aviso, e `disk is full` é disco cheio. |
| 6 | A chave de API está ausente; é preciso informar a chave. | `Missing` indica que algo necessário não foi enviado. |
